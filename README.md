
# SwagLabs Shopping

Este repositório contém a documentação produzida para o desafio de projeto **“O dia a dia de um QA: A prática de testes manuais funcionais”** da **DIO (Digital Innovation One)**, aplicada ao contexto de uma loja virtual fictícia chamada SwagLabs Shopping.

O objetivo principal do projeto é demonstrar, de forma prática, atividades do dia a dia de um QA, com foco em testes manuais funcionais, documentação e uso de ferramentas de apoio.

---


## Ferramentas e Metodologia

- **Metodologia:** Scrum
- **Ferramentas sugeridas:**
  - Jira (gestão de backlog, fluxo de trabalho e bugs)
  - Confluence (documentação)
  - Ferramenta de mind-map (XMind, MindMup, Miro, etc.)
- **Tipo de teste:** Manual, funcional, caixa-preta
- **Nível de teste:** Sistema e aceite (UAT)

---

## Documentação do Projeto

A documentação desenvolvida ao longo do projeto foi gerada em formato PDF, conforme as orientações do desafio da DIO, e está organizada na pasta  ``` docs``` deste repositório.

Os documentos apresentam, de forma estruturada, o fluxo de trabalho adotado, as User Stories e os casos de testes produzidos durante a prática de QA.
